## Prerequisites

Before starting, ensure you have:

<Steps>
  <Step title="Install adbcli">
    Follow the [installation guide](/installation) to install adbcli via cargo.
  </Step>
  
  <Step title="Install an ADBC driver">
    Install at least one ADBC driver. For this guide, we'll use DuckDB:

    ```bash
    dbc install duckdb
    ```
  </Step>
</Steps>

## Interactive Mode

The interactive mode provides a SQL shell with syntax highlighting, command history, and multiline input support.

### Start the Shell

Connect to DuckDB in-memory:

```bash
adbcli --driver duckdb
```

You'll see a prompt:

```
>
```

### Execute Your First Query

Let's create a table from a remote CSV file:

```sql
CREATE TABLE penguins AS FROM 'https://blobs.duckdb.org/data/penguins.csv';
```

<CodeGroup>
```sql Input
CREATE TABLE penguins AS FROM 'https://blobs.duckdb.org/data/penguins.csv';
```

```text Output

│ Count │
├───────┤
│ 344   │
└───────┘
```
</CodeGroup>

### Query the Data

Now let's query the table. adbcli supports multiline input - just continue typing on new lines:

```sql
SELECT *
FROM penguins
LIMIT 5;
```

### Exit the Shell

Press `Ctrl+C` twice to exit the interactive shell.

<Note>
  A single `Ctrl+C` will cancel the current input. Press it again to exit.
</Note>

## Non-Interactive Mode

adbcli also supports non-interactive usage for scripting and automation.

### Execute a Direct Query

Run a query and immediately exit:

```bash
adbcli --driver duckdb --query "SELECT 42 AS the_answer"
```

### Execute from a File

Create a SQL file:

```sql select_example.sql
SELECT
  species,
  COUNT(*) AS count,
  AVG(body_mass_g) AS avg_mass
FROM 'https://blobs.duckdb.org/data/penguins.csv'
GROUP BY species
ORDER BY avg_mass DESC;
```

Execute it:

```bash
adbcli --driver duckdb --file select_example.sql
```

### Execute from stdin

Pipe a query from stdin:

```bash
echo "SELECT 42 AS the_answer" | adbcli --driver duckdb
```

## Export Results

adbcli can export query results to various file formats.

<Tabs>
  <Tab title="JSON">
    Export to JSON format:

    ```bash
    adbcli --driver duckdb \
      --query "SELECT 42 AS the_answer" \
      --output result.json
    ```

    The file will contain Arrow JSON format data.
  </Tab>
  
  <Tab title="CSV">
    Export to CSV format:

    ```bash
    adbcli --driver duckdb \
      --query "SELECT 42 AS the_answer" \
      --output result.csv
    ```

    Standard comma-separated values format.
  </Tab>
  
  <Tab title="Arrow IPC">
    Export to Arrow IPC (feather) format:

    ```bash
    adbcli --driver duckdb \
      --query "SELECT 42 AS the_answer" \
      --output result.arrow
    ```

    This is the most efficient format for Arrow data.
  </Tab>
</Tabs>

<Note>
  The output format is automatically detected from the file extension. The `--output` flag cannot be used in interactive mode.
</Note>

## Connect to Other Databases

### DuckDB with a File

Connect to a persistent DuckDB database:

```bash
adbcli --driver duckdb --option path=mydatabase.db
```

### PostgreSQL

Connect to PostgreSQL with connection details:

```bash
adbcli --driver postgresql \
  --uri "postgresql://localhost:5432/mydb" \
  --username myuser \
  --password mypassword
```

### Driver-Specific Options

Pass driver-specific options using `--option`:

```bash
adbcli --driver duckdb \
  --option "access_mode=read_only" \
  --option "threads=4"
```

<Note>
  Options use the format `key=value`. You can specify multiple `--option` flags.
</Note>

## Customize Table Display

Change the table display style with `--mode`:

```bash
adbcli --driver duckdb \
  --query "SELECT 1, 2, 3" \
  --mode ascii-markdown
```

Available modes include `utf8-compact` (default), `ascii-markdown`, and others.
