The adbcli REPL (Read-Eval-Print Loop) provides an interactive SQL shell for exploring databases and running queries.

## Starting the REPL

Launch the interactive shell by connecting to a database without providing a query:

```bash
adbcli --driver duckdb
```

You'll see the SQL prompt:

```
> 
```

<Tip>
The REPL automatically starts when you don't provide `--query`, `--file`, or pipe data through stdin.
</Tip>

## Basic Usage

### Single-Line Queries

Type a SQL query and press Enter. End queries with a semicolon:

```sql
> SELECT 42 AS answer;
┌────────┐
│ answer │
├────────┤
│ 42     │
└────────┘
```

### Multi-Line Queries

For complex queries, press Enter without a semicolon to continue on the next line:

```sql
> SELECT 
.   species,
.   COUNT(*) as count
. FROM penguins
. GROUP BY species;
┌───────────┬───────┐
│ species   │ count │
├───────────┼───────┤
│ Adelie    │ 152   │
│ Gentoo    │ 124   │
│ Chinstrap │ 68    │
└───────────┴───────┘
```

Notice the `.` continuation prompt for multi-line input.

<Note>
Queries are executed when you type a semicolon (`;`) at the end of a line. Without a semicolon, pressing Enter continues to the next line.
</Note>

## Features

### Syntax Highlighting

The REPL includes real-time SQL syntax highlighting powered by Syntect, making queries easier to read and write:

- **Keywords** are highlighted (SELECT, FROM, WHERE, etc.)
- **Strings** are displayed in distinct colors
- **Numbers** and **operators** are visually distinguished

### Query Validation

The REPL validates query completeness:

- Complete queries (ending with `;`) execute immediately
- Incomplete queries wait for additional input
- Empty lines are ignored

### Command History

Navigate through previously executed queries:

<Steps>
  <Step title="Navigate History">
    Use **Up Arrow** (↑) and **Down Arrow** (↓) to browse command history
  </Step>
  <Step title="Search History">
    Press **Ctrl+R** to search through previous commands
  </Step>
  <Step title="Edit Previous">
    Recall a command and edit before re-executing
  </Step>
</Steps>

### History Search

Press **Ctrl+R** to enter search mode:

```
(search: SELECT) SELECT * FROM users WHERE active = true;
```

Type to filter history entries, then press Enter to execute or Esc to edit.

## Table Display Modes

Customize how results are displayed with the `--mode` flag:

```bash
adbcli --driver duckdb --mode ascii-markdown
```

## Exiting the REPL

<Tabs>
  <Tab title="Ctrl+D">
    Press **Ctrl+D** to exit cleanly:

    ```sql
    > SELECT * FROM data;
    [results displayed]
    > ^D
    $
    ```

    This is the recommended way to exit.
  </Tab>

  <Tab title="Ctrl+C">
    Press **Ctrl+C** once to cancel the current input:

    ```sql
    > SELECT * FROM 
    . ^C
    > 
    ```

    Press **Ctrl+C** twice quickly to exit the REPL:

    ```sql
    > ^C^C
    $
    ```
  </Tab>
</Tabs>

    ```sql
    │ Chinstrap │
    ```

  <Step title="Aggregate Analysis">

    ```sql
    │ Chinstrap │ 3733.088235294118│ 68    │

    ```
  </Step>

## Error Handling

The REPL displays errors and continues running:

```sql
> SELECT * FROM nonexistent_table;
Catalog Error: Table with name nonexistent_table does not exist!
> 
```

You can immediately try another query without restarting.

<Warning>
**SQL errors don't exit the REPL**

Syntax errors, missing tables, or constraint violations display an error message but keep the REPL active. This lets you correct mistakes and continue working.
</Warning>

## Working with Different Databases

<CodeGroup>
```bash PostgreSQL
adbcli --driver postgresql \
  --uri postgresql://postgres:password@localhost:5432/mydb
```

```bash MySQL
adbcli --driver mysql \
  --uri 'root:password@tcp(localhost:3306)/mydb'
```

```bash Snowflake
adbcli --driver snowflake \
  --uri snowflake://user:pwd@account/DB/SCHEMA?warehouse=WH
```

```bash BigQuery
adbcli --driver bigquery \
  --option adbc.bigquery.sql.project_id=my-project \
  --option adbc.bigquery.sql.dataset_id=analytics
```
</CodeGroup>

All REPL features work consistently across different database drivers.

## Advanced Configuration

### Custom Display Mode

```bash
# Use markdown-style tables
adbcli --driver duckdb --mode ascii-markdown

# Minimal output
adbcli --driver duckdb --mode nothing

# Full ASCII borders
adbcli --driver duckdb --mode ascii-full
```

### Connecting with Options

```bash
# DuckDB with custom settings
adbcli --driver duckdb \
  --option path=analytics.db \
  --option threads=8 \
  --option memory_limit='8GB'

# PostgreSQL with specific database
adbcli --driver postgresql \
  --uri postgresql://localhost/production \
  --username analyst \
  --password $(cat ~/.db_password)
```

## Limitations

<Warning>
**The `--output` flag is not available in interactive mode.**

To export query results to a file, use the `--query` flag instead:

```bash
# ❌ Not supported
adbcli --driver duckdb --output results.json

# ✅ Use non-interactive mode
adbcli --driver duckdb \
  --query "SELECT * FROM data" \
  --output results.json
```
</Warning>

Other considerations:

- The REPL prints results to stdout only
- Very large result sets may overflow the terminal
- Binary data displays as escaped strings
- Schema information is not shown (use `DESCRIBE` or similar SQL commands)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Query not executing" icon="circle-question">
    Ensure your query ends with a semicolon (`;`):
    
    ```sql
    > SELECT * FROM users;  ← Semicolon required
    ```
    
    Without the semicolon, press Enter to continue on the next line.
  </Accordion>

  <Accordion title="Cannot exit with Ctrl+C" icon="door-open">
    Press **Ctrl+C** twice in quick succession, or use **Ctrl+D** to exit immediately:
    
    ```
    > ^C^C
    ```
    
    The first Ctrl+C cancels input; the second exits the REPL.
  </Accordion>

  <Accordion title="Garbled output" icon="text-slash">
    Your terminal may not support UTF-8 characters. Try ASCII mode:
    
    ```bash
    adbcli --driver duckdb --mode ascii-full
    ```
  </Accordion>

  <Accordion title="Connection errors" icon="plug">
    Verify database connection before entering REPL:
    
    ```bash
    # Test with a simple query first
    adbcli --driver duckdb --query "SELECT 1"
    
    # Then start REPL
    adbcli --driver duckdb
    ```
  </Accordion>
</AccordionGroup>
