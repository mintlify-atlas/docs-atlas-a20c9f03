adbcli can connect to any database with a compatible ADBC driver. This guide covers connection patterns for popular databases.

## Connection Basics

All database connections use the same basic pattern:

```bash
adbcli --driver <driver_name> [OPTIONS]
```

<Note>
Before connecting to a database, install the appropriate ADBC driver using [dbc](https://docs.columnar.tech/dbc/).
</Note>

## Connection Options

adbcli supports several connection parameters:

<AccordionGroup>
  <Accordion title="--driver" icon="puzzle-piece">
    **Required.** The name of the ADBC driver to use.
    
    ```bash
    --driver duckdb
    ```
  </Accordion>

  <Accordion title="--uri" icon="link">
    Database connection URI. Format varies by driver.
    
    ```bash
    --uri postgresql://user:pass@host:5432/database
    ```
  </Accordion>

  <Accordion title="--username" icon="user">
    Database username for authentication.
    
    ```bash
    --username admin
    ```
  </Accordion>

  <Accordion title="--password" icon="lock">
    Database password for authentication.
    
    ```bash
    --password secret123
    ```
  </Accordion>

  <Accordion title="--option" icon="gear">
    Driver-specific options in `key=value` format. Can be specified multiple times.
    
    ```bash
    --option path=data.db --option read_only=true
    ```
  </Accordion>
</AccordionGroup>

## Popular Databases

<Tabs>
  <Tab title="DuckDB">
    DuckDB is a fast in-process analytical database.

    ### Installation

    ```bash
    dbc install duckdb
    ```

    ### In-Memory

    ```bash
    adbcli --driver duckdb
    ```

    ### Persistent Database

    ```bash
    adbcli --driver duckdb --option path=data.db
    ```

    ### MotherDuck

    Connect to MotherDuck cloud service:

    ```bash
    adbcli --driver duckdb --option path=md:sample_data
    ```
  </Tab>

  <Tab title="PostgreSQL">
    PostgreSQL and PostgreSQL-compatible databases.

    ### Installation

    ```bash
    dbc install postgresql
    ```

    ### Standard Connection

    ```bash
    adbcli --driver postgresql --uri postgresql://postgres:password@localhost:5432/postgres
    ```

    ### Compatible Databases

    The PostgreSQL driver also works with:

    **Neon**
    ```bash
    adbcli --driver postgresql --uri postgresql://cloud_admin:cloud_admin@localhost:55433/postgres
    ```

    **TimescaleDB**
    ```bash
    adbcli --driver postgresql --uri postgresql://postgres:password@localhost:5432/postgres
    ```

    **YugabyteDB**
    ```bash
    adbcli --driver postgresql --uri postgresql://yugabyte@localhost:5433/yugabyte
    ```

    **Yellowbrick**
    ```bash
    adbcli --driver postgresql --uri postgresql://ybdadmin:ybdadmin@localhost:5432/yellowbrick
    ```
  </Tab>

  <Tab title="MySQL">

## Troubleshooting

<Warning>
If you encounter connection errors, verify:
- The ADBC driver is installed with `dbc list`
- The database is running and accessible
- Connection credentials are correct
- Required driver-specific options are provided
</Warning>

### Option Format

Driver options must follow the `key=value` format:

```bash
# ✅ Correct
--option path=data.db

# ❌ Incorrect (missing equals sign)
--option path data.db

# ❌ Incorrect (empty key)
--option =value
```

Multiple options can be specified:

```bash
adbcli --driver duckdb \
  --option path=data.db \
  --option read_only=true \
  --option threads=4
```
